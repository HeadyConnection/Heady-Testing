🔴 CRITICAL FINDINGS — Root Cause Analysis

Root Cause #1: MCP→Localhost Architecture Gap

The MCP server (v5.0.0, Cloud Run) has 55 tools that all proxy to localhost:33xx endpoints. These only exist inside the main Cloud Run container — but the MCP server runs as a separate service. This is why every tool call fails.

Fix: Route MCP tools to Cloud Run production URL or internal service mesh, not localhost.

Root Cause #2: Cloud Run 404 — No Health Route

heady-production-uc.a.run.app/health returns 404. The PM2 container is either not starting cleanly (merge conflicts block CI) or the health endpoint isn't registered at the container's root.

Fix: Resolve .env.example merge conflicts → fix CI → redeploy container with proper health routes.

Root Cause #3: HCFP Can't Self-Heal

The auto-success pipeline (hc_auto_success.js) runs every 15 min but depends on the container being healthy. Phase 0 φ-Preflight finds merge conflicts and exits. The self-healing system cannot heal itself.

Fix: Make HCFP externally runnable (GitHub Actions + Edge Worker fallback) independent of container health.

Root Cause #4: nginx Proxy Misconfiguration

Cloud Run services (headyio, headyconnection, headybuddy, headymcp, admin-ui) all return 404 through nginx/1.29.x. The domain router isn't matching these service names to the correct upstream.

Fix: Update domain-router.js service map and nginx upstream configs.
